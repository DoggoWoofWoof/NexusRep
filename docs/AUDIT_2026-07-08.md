## Findings

### P0/P1 - Production `next start` is broken

`npm run build` succeeds, but `npm run start -- -p <port>` crashes with:

```text
TypeError: routesManifest.dataRoutes is not iterable
```

The generated `.next/routes-manifest.json` is missing route arrays that Next's
server path expects, including `dataRoutes`, `dynamicRoutes`, and `staticRoutes`.
Because the Playwright web server command is `npm run build && npm run start`,
the official `npm run e2e` path is blocked by the same issue.

Recommended fix: investigate the Next 15.1.6 output/config interaction, then
make production start boot cleanly before treating E2E as hermetic.

### P1 - Default E2E is flaky/blocked even though serial dev E2E passes

The full serial suite passes against a dev server, but the configured default
E2E path is fragile:

- `fullyParallel: true` caused timing failures under dev/server compile load.
- The configured production web server cannot boot because of the `next start`
  issue above.
- Chromium was missing initially and required `npx playwright install chromium`.

Recommended fix: first fix production start, then re-run default parallel E2E.
If parallel failures remain, isolate shared app state or run the lifecycle suite
serially while keeping independent visual tests parallel.

### P1 - Lint script is not a real CI check

`npm run lint` invokes `next lint`, which is deprecated/removed enough in this
setup that it prompts interactively instead of producing lint results.

Recommended fix: add a normal ESLint config/script, for example an `eslint`
script that runs non-interactively over `src`, `tests`, and `e2e`.

### P1/P2 - Dependency audit reports critical issues

`npm audit --audit-level=low` reports 11 vulnerabilities:

- Critical `protobufjs` path via `@xenova/transformers` / `onnxruntime-web`.
- Moderate `esbuild` path via Vite/Vitest.
- Moderate `postcss` path via Next.
- Several suggested `npm audit fix --force` resolutions are breaking or
  suspicious for this app, so they were not applied during the audit.

Recommended fix: upgrade deliberately, starting with the embeddings stack and
test runner. Avoid blind force upgrades.

### P2 - "Clean" demos can inherit durable local state

`.env.local` sets the app to the Postgres/PGlite data driver locally. That means
manual demos can carry forward old coaching rules, session artifacts, and other
state unless the environment explicitly overrides to memory or resets storage.

Recommended fix: make E2E/recording/demo scripts force
`NEXUSREP_DATA_DRIVER=memory`, and add a reset path for durable local demos.

### P2 - Studio readiness numbers conflict

The Studio setup surface can show conflicting readiness signals at the same time:

- Header: `100% ready - 0 items left`.
- Sidebar badge: `AI Rep 68%`.
- Setup chat: `0 of 9 answered`.
- Readiness review: `NEEDS INPUT`.

Recommended fix: consolidate these statuses behind one readiness source of truth
and make the Setup Assistant answered-count derive from the same model.

### P2 - HCP invite briefly first-paints generic copy

The HCP invite can initially render generic "this therapy" copy before the brand
profile API resolves and replaces it with Milvexian copy. This is a doctor-facing
polish and trust issue.

Recommended fix: show a small doctor-safe loading state until the brand context
is resolved, or server-render the brand profile for `/hcp`.

### P2 - Newly created live session review can be empty

Seeded session review opens with transcript/evidence. However, after creating a
live HCP session, the Sessions list can show a row with a question count/status
while the detail review opens with "This session has no recorded turns yet."

Recommended fix: trace the live `/hcp` session id through turn persistence and
the sessions repository. The list row and detail page should read the same turn
set.

### P3 - Visual regression coverage is narrower than the brief requires

Visual tests currently cover overview, A/V spike idle, and HCP invite. The brief
expects visual coverage for major screens and issues like hidden tabs, cramped
tables, clipped headers, overlapping controls, and internal terms in doctor view.

Recommended fix: add visual snapshots for Build, Training, Audience, Launch,
Sessions, Session Review, Analytics, Follow-ups, and an active HCP conversation.

### P3 - Live 3D avatar verification remains guarded

The 3D avatar test remains guarded/skipped unless explicit verification env vars
are enabled. It was not treated as verified in this audit.

Recommended fix: run a dedicated human-visible Chrome pass for the 3D avatar and
document expected first-load/network behavior separately from the default mock
path.
